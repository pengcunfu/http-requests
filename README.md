# HTTP 请求工具 - 专业版

一个功能丰富、界面美观的HTTP请求测试工具，基于PySide6开发。

## 功能特性

### 🚀 核心功能
- **多种HTTP方法支持**: GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS
- **异步请求处理**: 非阻塞UI，支持请求取消
- **智能URL处理**: 自动添加协议前缀
- **灵活的请求头管理**: 支持启用/禁用、添加/删除
- **多格式请求体**: 支持JSON、XML、纯文本等
- **详细响应信息**: 状态码、响应时间、数据大小

### 📊 界面特性
- **现代化UI设计**: 美观的界面和配色方案
- **分割窗口布局**: 左侧历史记录，右侧主功能区
- **标签页组织**: 请求头、请求体、响应分别管理
- **状态栏显示**: 实时显示操作状态和进度
- **响应式布局**: 适应不同窗口大小

### 💾 数据管理
- **请求历史记录**: 自动保存最近50次请求
- **配置保存/加载**: 支持导出和导入请求配置
- **JSON格式化**: 一键美化JSON数据
- **智能错误提示**: 详细的错误信息和解决建议

### ⚙️ 高级功能
- **超时设置**: 可配置请求超时时间(1-300秒)
- **请求头模板**: 预设常用请求头
- **历史记录管理**: 支持删除单个或清空全部
- **状态码颜色标识**: 不同状态码用不同颜色显示

## 安装和运行

### 方式一：下载预编译版本（推荐）
1. 访问 [Releases页面](https://github.com/yourusername/http-requests/releases)
2. 下载对应平台的压缩包：
   - Windows: `HTTP-Requests-Tool-Windows.zip`
   - Linux: `HTTP-Requests-Tool-Linux.tar.gz`
   - macOS: `HTTP-Requests-Tool-macOS.tar.gz`
3. 解压后直接运行可执行文件

### 方式二：从源码运行
#### 环境要求
- Python 3.9+
- PySide6
- requests

#### 安装依赖
```bash
pip install -r requirements.txt
```

#### 运行程序
```bash
python main.py
```

或者在Windows上双击 `run.bat` 文件。

## 使用指南

### 基本使用
1. **选择HTTP方法**: 从下拉菜单选择GET、POST等
2. **输入URL**: 在URL输入框中输入目标地址
3. **设置请求头**: 在"请求头"标签页中添加或修改请求头
4. **编写请求体**: 在"请求体"标签页中输入数据(POST/PUT等需要)
5. **发送请求**: 点击"发送请求"按钮
6. **查看响应**: 在"响应"标签页中查看结果

### 高级功能

#### 请求头管理
- 点击"添加请求头"按钮新增请求头
- 使用复选框启用/禁用特定请求头
- 选中行后点击"删除选中"移除请求头

#### 历史记录
- 左侧面板显示最近的请求历史
- 点击历史记录项可快速加载之前的请求
- 支持删除单个记录或清空全部历史

#### 配置管理
- 菜单栏 → 文件 → 保存请求: 导出当前请求配置
- 菜单栏 → 文件 → 加载请求: 导入之前保存的配置

#### JSON格式化
- 菜单栏 → 工具 → 格式化JSON: 美化当前标签页的JSON
- 响应页面的"格式化JSON"按钮: 美化响应中的JSON数据

### 响应信息显示
- **状态码**: 用颜色标识(绿色=成功，橙色=客户端错误，红色=服务器错误)
- **响应时间**: 以毫秒为单位显示

- **响应体**: 自动格式化JSON，保持其他格式原样

## 故障排除

### 常见问题

1. **程序无法启动**
   - 检查Python版本是否为3.7+
   - 确认已安装所有依赖: `pip install -r requirements.txt`

2. **请求失败**
   - 检查URL格式是否正确
   - 确认网络连接正常
   - 查看错误提示中的具体原因

3. **界面显示异常**
   - 尝试调整窗口大小
   - 重启程序

4. **历史记录丢失**
   - 检查用户主目录是否有写入权限
   - 确认 `.http_client_history.json` 文件是否存在
